# 🧬 evolab — 로또 전략 진화 실험실

"자유 탐색 → 성과 측정 → 좋은 전략 공유 → 진화" 루프의 **뼈대**.
Hall of Fame + 섬(Island) 모델로 설계됨.

## 파일 구조

| 파일 | 역할 |
|------|------|
| `core.py` | 데이터 로딩 · 당첨 채점 · ROI/적합도 계산 (순수 결정적 함수, LLM 무관) |
| `strategies.py` | 전략 기본 클래스 + 샘플 3종. 모든 전략은 **산술식(formula) + 사유(rationale) 필수** |
| `explorer.py` | 🤖 LLM 탐색가 — `claude -p`로 새 전략 생성 + **샌드박스 검증 게이트** |
| `engine.py` | 백테스트 · 명예의 전당 · 예측 원장 · 진화 사이클 |
| `hall_of_fame.json` | 🏆 공유 풀 — 성과 좋은 전략이 승격되어 저장됨 (자동 생성) |
| `evolved_strategies.json` | 🧬 LLM이 만든 진화 전략 누적 저장 (코드 포함, 자동 생성) |
| `ledger.jsonl` | 📒 회차×전략별 (사유·번호·결과) 누적 원장 (자동 생성) |
| `picks.json` | 🎯 "나또 공식 픽" — 회차당 주간 2세트(자동) + 온디맨드 추가 픽. 선정사유·결과·피드백 포함 (자동 생성) |
| `dashboard_data.json` | index.html "🎯 나또 공식 픽" / "🧬 evolab 진화 추천" 카드가 fetch (자동 생성) |

## 실행

```bash
cd evolab
python engine.py            # 결정적 사이클: 채점 → 평가 → 선발 → 예측 → 주간 공식 픽 2세트 자동 기록
python engine.py --llm      # + 단일 LLM(claude -p) 탐색가가 새 전략 2개 생성·검증·누적
python engine.py --llm -n 4 # 새 전략 4개 제안
python engine.py --islands  # 🌊 섬 모델: 4개 탐색가가 서로 다른 관점으로 동시 탐색
python engine.py --extra 3  # 주간 자동 픽과 별개로, 다음 회차 추가 픽 3세트를 온디맨드 생성
```

주간 자동 실행(Windows): `evolab/run_weekly.ps1` — 수집 → 진화(섬, 공식 픽 2세트 포함) → 로그.
작업 스케줄러 등록법은 스크립트 상단 주석 참고. (CI는 `claude -p` 불가 → LLM 진화는 로컬에서)

## 🎯 나또 공식 픽 (주간 2세트 + 추가 픽)

매 사이클(주간 자동 실행 포함)마다 명예의 전당 적합도 상위 2개 전략으로 **공식 픽 2세트**를
`picks.json`에 자동 기록한다(회차당 1번, 재실행해도 중복 안 됨). 각 세트는 선정 사유(`basis`,
어떤 전략·산식을 썼는지)를 포함하고, 다음 사이클에서 그 회차 실제 결과가 나오면 자동으로
채점되어 `result`(일치 번호·등수)와 `feedback`(정직한 해석 — 당첨/미당첨 모두 0.8개 기대값
기준 설명)이 채워진다.

추가 세트는 **자동으로 늘지 않는다** — 대시보드 또는 채팅 요청이 있을 때만
`python engine.py --extra N` 으로 그 시점의 명예의 전당 순위를 기반으로 생성되며, 이번 회차에
이미 공식 픽에 쓰인 전략은 제외된다. 대시보드의 "🎯 나또 공식 픽" 카드가 `weekly_picks`/
`extra_picks`를 각각 보여준다.

## LLM 탐색가 (explorer.py)

`claude -p`에게 명예의 전당 상위 전략(산술식·사유·적합도)을 보여주고 **더 나은
산술식+사유의 새 전략 코드**를 JSON으로 받는다. 받은 코드는 두 단계로 검증:

1. **정적 검사** — `import`/`os`/`eval`/`__…__` 등 위험 토큰 포함 시 거부
2. **샌드박스 실행** — 제한된 builtins + (random·math·Counter·past)만 주입해 실제
   `generate()`를 최근 5회차에 돌려 6개 유효 번호(1~45, 중복 없음)를 내는지 검증

통과한 전략만 `evolved_strategies.json`에 누적되어 다음 사이클부터 시드와 함께 경쟁한다.
> ⚠️ 샌드박스는 '합리적' 수준이지 완벽하지 않다. 신뢰 가능한 Claude 출력 + 로컬 개인용 전제.
> 낯선 LLM/원격 실행 시 컨테이너 격리 권장.

## 한 사이클이 하는 일 (`run_cycle`)

1. **채점** — `ledger.jsonl`의 pending 예측 중 이미 추첨된 회차를 실제 결과로 채점
2. **모집단** — 시드 전략 + (다음 단계) LLM이 제안한 새 전략
3. **백테스트** — 각 전략을 최근 100회에 굴려 ROI·적중분포·적합도 측정 (미래참조 차단)
4. **공유** — 적합도 상위 전략을 명예의 전당에 승격/저장
5. **예측** — 상위 5개 전략이 다음 회차 번호+사유를 원장에 pending으로 기록

## 적합도 (정직한 설계)

```
fitness = 0.35·norm(ROI) + 0.40·비인기도 + 0.25·통계건전성
```
- **비인기도(unpopularity)**: 남들이 안 고르는 조합 → 1·2·3등 당첨 시 분배 회피. **수학적으로 실재하는 유일한 엣지.**
- **건전성(soundness)**: 합계·홀짝·분산이 정상 범위인가.
- **ROI**: 관측 수익률. ⚠️ **대부분 분산(운)** — 기대 적중 개수는 어떤 전략이든 0.8개로 동일하다. 그래서 가중치를 보조 수준으로 둠.
